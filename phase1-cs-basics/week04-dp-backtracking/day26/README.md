- **할 일**: `day26/` 안에 `pruning_notes.md`(또는 주석)를 만든다. 다음 중 **2가지 이상**을 예시로 적는다.
  - **조합/합**: "남은 원소를 다 더해도 target에 못 미침" → early return.
  - **정렬 후**: 정렬해 두고 "현재 합 + (남은 개수 × 최소 원소) > target"이면 더 볼 필요 없음.

## 📌 참고 (복습할 개념)

- **가지치기(Pruning)**: "이 경로를 더 따라가도 정답이 나올 수 없다"를 **일찍 판단**해서 재귀를 끊는 것이다. 예: 조합/합 문제에서 "남은 원소를 전부 더해도 target에 못 미친다"면 그 분기는 더 볼 필요가 없다. **정렬**해 두면 "현재 합 + (남은 개수 × 최소 원소) > target"이면 더 키워도 target을 넘어가므로 early return할 수 있다.
- **N-Queens**: 행마다 퀸을 하나씩 두고, **열**이 겹치는지, **대각선**(row-col 또는 row+col이 같은지)이 겹치는지 체크해서, 유효한 칸에서만 재귀를 타면 불가능한 배치를 일찍 잘라 낸다.
