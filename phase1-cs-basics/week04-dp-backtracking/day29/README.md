# Day 29: 모의 인터뷰 — LeetCode Easy 3문제 (45분)

## 🎯 오늘의 목표

- **타이머 45분** 안에 **Easy 3문제**를 풀고, 접근법·복잡도·엣지케이스를 말로 정리할 수 있게 한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 모의 인터뷰 세션 진행

- **할 일**: 아래를 **순서대로** 진행한다.
  1. **타이머 45분** 설정 (알람 또는 타이머 앱).
  2. **NeetCode 또는 LeetCode**에서 **Easy 난이도 3문제**를 미리 정해 둔다 (예: Array 1, String 1, Hash 1).
  3. 45분 동안 **인터뷰처럼** 말하면서 풀어 본다. "이건 해시로 보겠습니다. 시간복잡도 O(n) 목표입니다." 등.
  4. 시간 내에 3문제를 **최대한** 풀고, 못 푼 문제는 "어디까지 생각했는지" 2~3문장 메모.
- **완료 조건**: 45분이 끝날 때까지 3문제 모두 시도했고, 풀었으면 코드가 동작하고, 못 풀었으면 막힌 지점을 적어 뒀다.

### 2. 풀이 코드·복기 저장

- **할 일**: 풀이 코드를 **`phase1-cs-basics/review/day29-mock/`** 또는 **`phase1-cs-basics/mock-interviews/day29/`** 에 저장한다. 파일명 예: `p1_두_줄_요약.py`, `p2_...`, `p3_...`. **복기 노트**를 같은 폴더에 `review.md`로 만든다. 다음을 포함한다.
  - **문제별**: 접근법 한 줄, 시간/공간복잡도, 엣지케이스. 못 푼 문제는 "어디서 막혔는지, 다음에 할 개선(예: 어떤 패턴 복습)".
- **완료 조건**: 3문제 분량의 코드(또는 부분 코드)와 복기 노트가 한 폴더에 있다.

### 3. (선택) 타이머 없이 못 푼 문제 재도전

- **할 일**: 시간 초과로 못 푼 문제가 있으면, 타이머 없이 **1문제**만 다시 풀어 본다. 풀이를 위 폴더에 추가하고, 복기 노트에 "재도전 결과" 한 줄 추가.
- **완료 조건**: 재도전한 문제를 풀었으면 코드와 "왜 처음에 막혔는지" 1줄 적어 뒀다.
