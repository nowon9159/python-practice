# Day 28: Phase 1 종합 복습 — 패턴 카탈로그 정리

## 🎯 오늘의 목표

- Phase 1 **전체 패턴**을 한 문서로 카탈로그화하고, 무작위로 5개 이상 백지 설명이 가능한지 확인한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 패턴 카탈로그 문서 작성

- **할 일**: `phase1-cs-basics/review/`(또는 `phase1-cs-basics/` 아래)에 **`pattern-catalog-phase1.md`** 를 만든다. 다음을 포함한다.
  - **Week 1**: Two Pointers, Sliding Window, Hash, Stack, Monotonic Stack — 각각 "언제 쓰는가, 대표 문제 1개, 시간복잡도 한 줄".
  - **Week 2**: Linked List, Fast/Slow, Binary Search(기본·변형·정답범위), Tree, BFS/DFS — 동일 형식.
  - **Week 3**: Heap, 인접 리스트/행렬, Island, 위상정렬, Union-Find, Dijkstra — 동일 형식.
  - **Week 4**: DP(Memo/Tab, 2D, Knapsack), Backtracking, Monotonic Deque — 동일 형식.
  - **(선택)** "문제 유형 → 추천 패턴" 매핑 테이블 한 블록.
- **완료 조건**: Phase 1에서 다룬 패턴이 **최소 15개 이상** 항목으로 정리되어 있고, "언제 쓰는가"가 한 줄씩 있다.

### 2. 취약 패턴 표시 후 보강

- **할 일**: 카탈로그에서 **자기 기준으로 취약한 패턴 2~3개**를 표시한다(예: ⚠️). 해당 패턴의 대표 문제를 **1문제씩** 다시 풀거나 해설을 보고 핵심만 3줄 요약해 카탈로그에 적어 둔다.
- **완료 조건**: 취약 패턴에 대해 "다시 풀었음" 또는 "핵심 요약"이 카탈로그 또는 별도 파일에 있다.

### 3. 무작위 패턴 5개 이상 백지 설명

- **할 일**: 카탈로그에서 **무작위로 5개** 패턴을 골라, 참고 없이 **1분 안에** 각 패턴을 말로 설명해 본다(또는 3~5문장으로 적어 본다). 막힌 것은 카탈로그로 복습.
- **완료 조건**: 5개 중 3개 이상을 막히지 않고 설명했다. 막힌 2개는 카탈로그로 채웠다.
