## ✅ 오늘 할 일 (순서대로)

### 1. Dijkstra 구현 (heap 기반)

- **할 일**: `day20/` 안에 `dijkstra.py`를 만든다.
  - **입력**: 그래프(인접 리스트: 노드 → (이웃, 가중치) 리스트), 출발 노드, 노드 개수 V.
  - **동작**: 거리 배열 dist, (거리, 노드)를 넣는 **min-heap**. 출발점 0, 나머지 무한(큰 수). heap에서 (d, u) 꺼내서, u를 통해 이웃 v로 가는 거리가 더 짧으면 갱신하고 heap에 (new_d, v) 추가. 이미 처리된 노드는 스킵(또는 dist보다 큰 d는 스킵).
  - **출력**: 출발점에서 각 노드까지 최단 거리 리스트 (또는 도달 불가 시 무한).
  - **가정**: 가중치 non-negative.
- **완료 조건**: 작은 그래프(예: 4~5노드)에 대해 손으로 구한 최단거리와 결과가 일치한다.

### 2. 서비스 메시 최단 경로 탐색기

- **할 일**: 같은 폴더에 `service_mesh_shortest_path.py`를 만든다.
  - **동작**: 노드 = 서비스 이름, 엣지 = 레이턴시(ms) 등. 서비스 간 레이턴시가 주어졌을 때 **한 서비스에서 다른 서비스까지의 최단 경로**(합 레이턴시 최소)를 **Dijkstra**로 구해 출력. (경로 복원: parent 배열 저장 후 역추적)
  - **입력**: 하드코딩 또는 파일/딕셔너리로 (서비스A, 서비스B, 레이턴시) 리스트.
  - **출력**: 출발 서비스 → 도착 서비스에 대한 최단 거리(및 경로).
- **완료 조건**: 예시 서비스 그래프 1개에 대해 Dijkstra로 최단 경로가 출력되고, 경로가 있으면 거리와 노드 순서가 맞다.

## 📌 참고 (복습할 개념)

- **Dijkstra**: **가중치가 음이 아닌** 그래프에서 한 출발점에서 모든 노드까지의 **최단 거리**를 구하는 알고리즘이다. **Relaxation**: "현재 dist[v]보다 dist[u] + 간선 가중치 w(u,v)가 더 작으면 dist[v]를 그 값으로 갱신"한다. **Heap**에 (거리, 노드)를 넣고, 가장 거리가 작은 노드를 꺼내서 그 노드를 경유하는 이웃들을 relaxation한다. 이미 더 짧은 거리로 처리된 노드는 스킵한다.
- **음수 간선**: 가중치에 음수가 있으면 Dijkstra는 잘못된 결과를 낼 수 있다. 그때는 **Bellman-Ford** 등 다른 알고리즘을 쓴다(Phase 3에서 다룸).
