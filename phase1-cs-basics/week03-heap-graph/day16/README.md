## ✅ 오늘 할 일 (순서대로)

### 1. 그래프 클래스 두 가지 구현

- **할 일**: `day16/` 안에 `graph_representation.py`를 만든다.
  - **인접 리스트**: `dict[int, list[int]]` 또는 `list[list[int]]` (노드 번호 0..V-1). `add_edge(u, v)` (무방향/방향 선택), `get_neighbors(v)`.
  - **인접 행렬**: `V×V` 2차원 리스트. `add_edge(u, v)`, `has_edge(u, v)` O(1).
  - **동일 입력**: 같은 엣지 집합을 두 표현에 넣고, `get_neighbors`/`has_edge` 결과가 일치하는지 확인.
- **완료 조건**: 두 표현 모두로 작은 그래프(예: 5노드)를 만들고, 엣지 추가·조회가 동작한다. 주석에 "인접 리스트 공간 O(V+E), 행렬 공간 O(V²), has_edge 행렬 O(1)" 등 적어 둔다.

### 2. 그래프 BFS/DFS 구현

- **할 일**: 같은 파일(또는 별도)에 **graph_bfs(graph, start)** 와 **graph_dfs(graph, start)** 를 구현한다. 방문 순서를 리스트로 반환. (재귀 DFS 시 재귀 한도 주의)
  - BFS: 큐, 방문 체크 세트
  - DFS: 스택 또는 재귀, 방문 체크
- **완료 조건**: 동일 그래프에 대해 BFS/DFS를 돌려 방문 순서를 출력했고, 예상과 맞다.

## 📌 참고 (복습할 개념)

- **인접 리스트**: 각 노드마다 "그 노드에서 나가는(또는 연결된) 이웃 목록"을 리스트로 둔다. **희소 그래프**(엣지가 적을 때)에 적합하고, 공간은 O(V+E)이다. 이웃 순회는 O(degree)라서 전체 BFS/DFS는 O(V+E)다.
- **인접 행렬**: V×V 2차원 배열에서 [u][v]=1(또는 가중치)이면 u→v 간선이 있다고 본다. **엣지 존재 여부**를 O(1)에 알 수 있지만 공간이 O(V²)라서 엣지가 많지 않으면 낭비다.
- **방문 체크**: BFS/DFS 모두 "이미 방문한 노드"를 set이나 배열로 표시하고, 큐/스택에 넣기 전에 체크해야 무한 루프를 막을 수 있다. **무방향**이면 (u,v) 추가 시 (v,u)도 추가하는 식으로 양쪽을 이어 준다.
