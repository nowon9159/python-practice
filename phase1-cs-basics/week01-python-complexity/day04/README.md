## 📌 참고 (복습할 개념)

- **해시(HashMap/HashSet)**: 키를 해시 함수로 "버킷 번호"로 바꿔 저장한다. 서로 다른 키가 같은 버킷에 들어가면 collision인데, chaining(리스트로 엮기) 등으로 처리한다. 평균적으로 삽입·조회·삭제가 O(1)이라, "빠르게 찾기"가 필요할 때 쓴다.
- **활용 상황**: (1) **카운팅** — 키에 원소, 값에 개수를 두면 빈도·Two Sum의 보수 등이 O(n)에 가능하다. (2) **존재 여부** — set이나 dict의 `in`으로 O(1) 확인. (3) **그룹핑** — 같은 키(예: 정렬된 문자열, 튜플)를 가진 것끼리 리스트로 묶을 때 dict를 쓴다.
