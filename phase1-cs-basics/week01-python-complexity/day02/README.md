## 📌 참고 (복습할 개념)

- **Big-O**: 입력 크기가 커질 때 시간·공간이 얼마나 늘어나는지 "최악 기준"으로 표기한다. 상수 배수는 무시하고, 식에서 가장 빨리 커지는 항(dominant term)만 남긴다. 예: 2n² + n → O(n²).
- **재귀**: 함수가 자기 자신을 호출하는 방식. 반드시 **종료 조건(base case)**이 있어야 하고, 그렇지 않으면 무한 호출이 된다. n번째 답을 "n-1, n-2의 결과로 표현"하는 식을 recurrence(점화식)라고 한다. 호출마다 스택 프레임이 쌓이므로 깊이가 크면 스택 오버플로우에 주의한다.
- **재귀→DP**: 같은 인자로 재귀가 여러 번 호출되면 중복 계산이 생긴다. **메모이제이션**은 한 번 계산한 결과를 캐시(딕셔너리 등)에 넣어 두고 다시 쓰는 것이고, **Tabulation**은 반복문으로 작은 인자부터 차례로 채워 올라가는(bottom-up) 방식이다. 둘 다 중복을 제거해 복잡도를 낮춘다.
