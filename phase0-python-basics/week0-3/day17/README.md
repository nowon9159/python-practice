# Day 17: [미니] 로그 파일 한 줄씩 읽어서 print

## 🎯 오늘의 목표

- **로그처럼 생긴 텍스트 파일**을 한 줄씩 읽어, **레벨**이나 **메시지**만 골라서 출력하는 스크립트를 만든다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 로그 형식 파일 만들기

- **할 일**: `day17/` 안에 `app.log` 파일을 만든다. 형식은 자유롭게, 예: `2024-01-01 10:00:00 [INFO] 서버 시작` 처럼 **날짜/시간 [레벨] 메시지** 형태로 3~5줄을 적어 둔다.
- **완료 조건**: app.log에 3줄 이상의 로그 형식 텍스트가 있다.

### 2. 한 줄씩 읽어서 레벨/메시지만 출력

- **할 일**: `day17/` 안에 `log_reader.py`를 만든다. `app.log`를 **open**하고 **for line in f** 로 한 줄씩 읽는다. 각 줄에서 (예: `[INFO]` 같은 단어가 있으면 그 부분만 찾거나, split으로 나눠서) **레벨**(INFO, ERROR 등)과 **메시지** 부분만 골라 **print** 한다. 정규식 없이 **split(), in, 인덱싱**만 써도 된다. 예: `if "[INFO]" in line: print("INFO:", line.split("]")[-1].strip())`
- **완료 조건**: app.log를 한 줄씩 읽어, 레벨 또는 메시지가 포함된 출력이 나온다.

---

## 📌 참고

- 로그 한 줄을 split(" ") 또는 split("]") 등으로 나눠서 필요한 부분만 골라 쓰면 된다.
- DevOps에서 보는 로그도 결국 "한 줄씩 읽어서 파싱"하는 흐름과 같다.
