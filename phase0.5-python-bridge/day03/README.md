# Day 3: None + if __name__ == "__main__"

## 🎯 오늘의 목표

- **None**이 "값이 없다"를 뜻한다는 걸 확인하고, **`if __name__ == "__main__":`** 으로 "직접 실행할 때만 실행되는 블록"을 써 본다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. None 확인하기

- **할 일**: `day03/` 안에 `none_basics.py`를 만든다.
  - **`d = {"a": 1}`** 에서 **`print(d.get("a"))`** → 1, **`print(d.get("b"))`** → None 이 나오는지 확인한다.
  - **None**은 "값이 없다"를 나타내는 객체라고 생각하면 된다. 나중에 Phase 1에서 **`Optional[str]`** 같은 타입은 "str 또는 None"이라는 뜻이다.
- **완료 조건**: `d.get("b")` 가 None을 반환하는 것을 확인했고, None이 "없음"을 뜻한다는 걸 한 줄이라도 메모해 뒀다.

### 2. if __name__ == "__main__" 쓰기

- **할 일**: 같은 폴더에 **`main_block.py`** 를 만든다.
  - **`if __name__ == "__main__":`** 아래에 **`print("이 파일을 직접 실행했을 때만 나옵니다.")`** 를 넣는다.
  - 터미널에서 **`uv run python main_block.py`** (또는 `python main_block.py`) 로 실행하면 문장이 출력된다.
  - 이 파일을 **다른 파일에서 import**하면 이 블록은 실행되지 않는다. 즉, "이 스크립트가 **직접 실행될 때만** 테스트나 실행 코드를 돌리려면 여기에 넣는다"는 뜻이다.
- **완료 조건**: `python main_block.py` 로 실행했을 때만 print가 나오는 것을 확인했다.

---

## 📌 참고

- **None**: Python에서 **"값이 없다"** 를 나타내는 객체다. `d.get("키")` 에서 키가 없으면 None이 반환된다. 타입 표기에서 **Optional[T]** 는 "T 타입이거나 None"이라는 뜻이다.
- **if __name__ == "__main__"**: Python이 이 파일을 **직접 실행**할 때는 **`__name__`** 이 **`"__main__"`** 이다. 다른 파일에서 **import**할 때는 `__name__` 이 그 모듈 이름이 되어 이 블록이 실행되지 않는다. 그래서 "테스트 코드나 실행 진입점"을 이 블록 안에 두면, import 시에는 안 돌고 직접 실행할 때만 돌린다.
