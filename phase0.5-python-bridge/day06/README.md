# Day 6: (선택) 데코레이터, 리스트 컴프리헨션, enumerate, with

## 🎯 오늘의 목표

- **데코레이터**, **리스트 컴프리헨션**, **enumerate**, **with** 개념을 한 줄씩 알아 두면 Phase 2·3에서 덜 낯설다. 여기서는 맛보기만 하고, 본격은 해당 Phase 과제를 풀면서 해도 된다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 네 가지 맛보기

- **할 일**: 아래 네 가지는 **Phase 1·2·3 과제를 풀면서** 자연스럽게 써 보면 된다. 미리 한 번만 들어 본다면:
  - **데코레이터**: 함수 위에 **`@함수명`** 을 쓰면, 그 함수가 "아래 함수를 감싼다"는 뜻이다. Phase 2에서 `@profile`, Phase 3에서 `@lru_cache` 를 쓸 때 나온다.
  - **리스트 컴프리헨션**: **`[x * 2 for x in [1, 2, 3]]`** → `[2, 4, 6]` 처럼, 리스트를 한 줄로 만드는 문법이다. for로 돌려서 append 하는 것과 같다.
  - **enumerate**: **`for i, x in enumerate(["a", "b", "c"]):`** 하면 `i` 에 0, 1, 2, `x` 에 "a", "b", "c" 가 들어간다. 인덱스와 값을 같이 쓸 때 편하다.
  - **with**: **`with open(...) as f:`** 는 "블록에 **들어갈 때** 파일을 열고, **나갈 때** 자동으로 닫는다"는 뜻이다. 나중에 Phase 2에서 소켓·락 같은 걸 쓸 때도 `with` 로 "준비 → 사용 → 정리" 패턴을 쓴다.
- **완료 조건**: (선택) 위 네 가지 중 하나라도 코드에 한 번 써 봤거나, "나중에 이런 게 나온다"고 한 줄 메모해 뒀다.

---

## 📌 참고

- **데코레이터**: "함수를 인자로 받아서, 그 함수를 감싼 새 함수를 반환하는 함수"를 **`@이름`** 으로 함수 위에 붙여 쓴다.
- **리스트 컴프리헨션**: **`[식 for 항목 in 반복가능한것]`** 형태로, 한 줄로 리스트를 만든다.
- **enumerate**: **`enumerate(리스트)`** 는 (인덱스, 값) 쌍을 순서대로 넘겨 준다.
- **with**: 블록 **진입 시** 준비(예: 파일 열기), **퇴장 시** 정리(예: 파일 닫기)를 보장하는 문법이다.
