# Day 77: 시스템 설계 심화 — CI/CD 파이프라인 아키텍처

## 🎯 오늘의 목표

CI/CD 파이프라인의 단계(빌드, 테스트, 배포), 트리거, 브랜치 전략, 실패 시 롤백·알림을 정리한 설계 문서를 작성하고 DevOps 연결고리를 Anki에 반영한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 파이프라인 단계 다이어그램

- **할 일**: `phase3-advanced/week11-12-final/day77/` 안에 `design.md`(또는 `cicd_design.md`)를 만든다. **파이프라인 단계 다이어그램**을 그린다. (lint → test → build → deploy 등. 병렬/순차, 캐시, 시크릿 관리 위치를 표시.) Jenkins/GitHub Actions/GitLab CI 중 하나와 개념 연결해 1~2문장으로 적는다.
- **완료 조건**: 문서에 단계별 흐름이 다이어그램 또는 리스트로 있고, 각 단계 역할이 드러난다.

### 2. 브랜치별 전략·실패 처리

- **할 일**: 같은 문서에 **브랜치별 전략**을 적는다. (예: main → prod, PR → staging, feature 브랜치 → 테스트만.) **실패 시 롤백·알림** 설계를 2~3문장으로 적는다.
- **완료 조건**: 브랜치별로 어떤 파이프라인이 돌고, 실패 시 어떻게 할지가 문서에 있다.

### 3. DevOps 연결·Anki

- **할 일**: 문서에 **DevOps 연결고리**를 한 줄씩 정리한다. (실무에서 쓰는 도구·개념.) Anki에 "CI/CD 아키텍처"(단계, 브랜치 전략, 롤백) 카드 추가.
- **완료 조건**: 설계 문서가 완성되었고, Anki 카드가 추가되어 있다.
