# Day 75: 시스템 설계 심화 — 대규모 검색 엔진 설계

## 🎯 오늘의 목표

대규모 검색 엔진의 요구사항·규모 추정을 하고, Crawler·Indexer·Query Service·Ranker 등 핵심 컴포넌트와 역색인·쿼리 흐름을 정리한 설계 문서를 작성한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 요구사항·규모 추정

- **할 일**: `phase3-advanced/week11-12-final/day75/` 안에 `design.md`(또는 `search_engine_design.md`)를 만든다. **요구사항**(검색 대상, QPS, 지연 시간 목표 등)과 **규모 추정**(일간 인덱싱량, 쿼리 수, 저장 용량 추정) 섹션을 적는다.
- **완료 조건**: 문서에 요구사항 3개 이상, 규모 추정 수치(예: 일 1B 문서, QPS 10k)가 적혀 있다.

### 2. High-level 다이어그램 + 데이터 모델

- **할 일**: 같은 문서에 **High-level 아키텍처 다이어그램**을 추가한다. (텍스트/ASCII 또는 Mermaid 등.) Crawler, Indexer, Query Service, Ranker, 저장소 역할을 표시. **데이터 모델**(문서, 역색인 구조 등)을 간단히 기술한다.
- **완료 조건**: 다이어그램으로 컴포넌트 간 흐름이 보이고, 역색인(inverted index)·쿼리 흐름을 문장으로 설명할 수 있다.

### 3. 핵심 컴포넌트·Anki

- **할 일**: 문서에 **핵심 컴포넌트 역할** 정리(각 2~3줄). 역색인, 분산 인덱스/샤딩, TF-IDF·PageRank 등 랭킹 개념을 요약. Anki에 "검색 엔진 설계 포인트"(역색인, 쿼리 흐름, 랭킹) 카드 추가.
- **완료 조건**: 설계 문서가 완성되었고, 역색인·쿼리 흐름을 말로 설명할 수 있으며 Anki에 반영했다.
