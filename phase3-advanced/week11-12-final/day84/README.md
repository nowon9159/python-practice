# Day 84: 최종 프로젝트 완성 + 코드 리뷰

## 🎯 오늘의 목표

Day 83 분산 태스크 큐에 에러 처리·재시도·(선택) 결과 조회를 넣어 완성하고, README와 코드 품질 체크리스트로 자기 코드 리뷰를 한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 기능 마무리

- **할 일**: `phase3-advanced/final-project/`(또는 day83에서 만든 폴더)의 태스크 큐에 **에러 처리**(태스크 실행 예외 처리), **재시도**(실패 시 N회 재시도), **(선택) 결과 조회**를 추가한다. 타임아웃이 필요하면 설정 가능하게 한다.
- **완료 조건**: 태스크 실패 시 예외가 잡히고, 재시도 동작(또는 스킵)이 확인된다. (선택) 결과를 조회할 수 있다.

### 2. README 작성

- **할 일**: 프로젝트 루트에 **README.md**를 만든다. **목적**(Celery 스타일 미니 태스크 큐), **설치·실행 방법**(uv/ pip, Broker 실행, Worker 실행, 클라이언트 예제), **아키텍처 요약**(Producer–Broker–Worker, 사용한 큐 종류)을 적는다.
- **완료 조건**: README만 보고 설치·실행이 가능하고, 아키텍처가 2~3문장으로 설명되어 있다.

### 3. 코드 품질 체크리스트로 자기 리뷰

- **할 일**: Day 74에서 만든 **코드 품질 체크리스트**를 열고, 이 프로젝트에 대해 자기 리뷰를 한다. (네이밍, 엣지케이스, 복잡도, 테스트 가능성 등.) 리뷰 결과를 `day84/review_notes.md` 또는 프로젝트 내 `REVIEW.md`에 5~10줄로 적는다.
- **완료 조건**: 프로젝트가 실행·테스트 가능한 상태이고, README와 주요 결정 사항이 문서화되었으며, 리뷰 메모가 저장되어 있다.
