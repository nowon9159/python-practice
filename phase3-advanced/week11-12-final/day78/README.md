# Day 78: 오픈소스 기여 — CPython or Ansible 이슈 탐색

## 🎯 오늘의 목표

CPython 또는 Ansible(또는 관심 오픈소스) 저장소의 기여 가이드를 읽고, Good first issue·문서 수정·작은 버그 수정 후보 이슈 2~3개를 선정해 분석한 뒤, 다음에 할 기여 후보를 정리한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 저장소 fork·기여 가이드 읽기

- **할 일**: **CPython** 또는 **Ansible** (또는 선택한 오픈소스) 저장소를 fork하고, **CONTRIBUTING.md**·기여 가이드 문서를 읽는다. 로컬 빌드/테스트 방법을 `phase3-advanced/week11-12-final/day78/` 안에 `contribution_guide.md`로 요약한다. (이슈 트래커 사용법, good first issue·documentation 라벨 설명 포함.)
- **완료 조건**: 기여 가이드 요약이 문서에 있고, 로컬 빌드/테스트 절차가 3단계 이상 적혀 있다.

### 2. 이슈 2~3개 선정·분석

- **할 일**: 이슈 트래커에서 **관심 이슈 2~3개**를 선정한다. (good first issue, documentation, 작은 버그 등.) `day78/issues.md`에 **이슈 번호·제목·선정 이유·재현 방법 또는 원인 추정**을 적는다.
- **완료 조건**: 2~3개 이슈에 대해 선정 이유와 (가능하면) 재현 방법 또는 원인 추정이 문서에 있다.

### 3. (선택) PR 시도·다음 기여 후보 정리

- **할 일**: (선택) 문서 수정 PR 또는 작은 패치를 시도해 본다. 하지 않더라도 **다음에 할 기여 후보**를 `issues.md` 또는 별도 섹션에 1~2문장으로 정리한다.
- **완료 조건**: 기여 가이드 정리가 완료되었고, 이슈 목록·선정 이유가 문서에 있으며, 다음에 할 기여 후보가 메모되어 있다.
