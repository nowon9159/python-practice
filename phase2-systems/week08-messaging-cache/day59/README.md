# Day 59: 설계 실습 — 쿠버네티스 스케줄러 단순화 구현

## 🎯 오늘의 목표

- **쿠버네티스 스케줄러** 개념(노드 선택, 리소스 고려)을 정리하고, **Pod 스펙 + 노드 목록 → 선택된 노드**를 반환하는 **단순화 스케줄러**를 구현한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 스케줄링 전략 정리

- **할 일**: `day59/` 안에 `scheduler_notes.md`를 만든다. **Pod**가 요청하는 CPU/메모리와 **노드**의 가용 자원을 비교한다. **First Fit**(첫 번째로 들어가는 노드), **Bin Packing**(가용 자원이 가장 적게 남는 노드) 등 1~2가지 전략을 2문장으로 적는다.
- **완료 조건**: "리소스 요청 vs 가용 자원, 노드 선택 규칙"이 문서에 있다.

### 2. 단순화 스케줄러 구현

- **할 일**: `day59/` 안에 `simple_scheduler.py`를 만든다. **입력**: Pod 스펙 리스트(예: `[{cpu: 1, mem: 2}, ...]`), 노드 목록(예: `[{name: "n1", cpu_avail: 4, mem_avail: 8}, ...]`). **출력**: 각 Pod에 대해 **배치할 노드 이름**(또는 None). 규칙: Pod의 cpu/mem 요청이 노드의 가용 자원 이하인 노드 중 하나를 선택 (First Fit 또는 점수 기반). 가용 자원은 배치 시 감소시키는 로직을 넣는다.
  - **완료 조건**: Pod 2~3개, 노드 2~3개로 실행해, 모든 Pod가 배치되거나 일부는 None(자원 부족)이 나온다.

### 3. Anki 카드 추가

- **할 일**: "스케줄러: 리소스 요청·가용 자원, 노드 선택 전략" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
