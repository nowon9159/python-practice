# Day 58: 설계 실습 — 분산 메시지 큐 설계

## 🎯 오늘의 목표

- **분산 메시지 큐**를 **요구사항 → 컴포넌트 → 파티션·오프셋** 순으로 설계 문서로 정리한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 요구사항·컴포넌트 다이어그램

- **할 일**: `day58/` 안에 `distributed_mq_design.md`를 만든다. **요구사항**: 처리량, 순서 보장 범위, 내구성 중 2~3개. **포넌트**: 브로커, 프로듀서, 컨슈머, (선택) 파티션. 다이어그램으로 브로커·토픽·파티션 관계를 그린다.
- **완료 조건**: 요구사항과 다이어그램이 문서에 있다.

### 2. 파티션·오프셋 설계

- **할 일**: 같은 문서에 **파티션**이 무엇인지(토픽을 나눈 단위), **오프셋**이 무엇인지(파티션 내 메시지 위치)를 2~3문장으로 적는다. Consumer group, offset commit을 한 줄이라도 적어 둔다.
- **완료 조건**: 파티션·오프셋의 역할이 설명되어 있다.

### 3. (선택) 단순화 브로커 코드

- **할 일**: Day 52 브로커를 확장해 "토픽 = 파티션 1개 리스트"로 두고, 오프셋(인덱스)으로 읽기하는 형태를 만들어 본다.
- **완료 조건**: 선택. 구현했다면 produce → consume(offset)이 동작한다.

### 4. Anki 카드 추가

- **할 일**: "분산 MQ: 파티션, 오프셋, 컨슈머 그룹" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
