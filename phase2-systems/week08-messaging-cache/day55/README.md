# Day 55: Consistent Hashing — 구현 + 노드 추가/제거 실험

## 🎯 오늘의 목표

- **Consistent Hashing**(해시 링, 가상 노드)을 정리하고, **노드 추가/제거** 시 어떤 키가 이동하는지 실험한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. Consistent Hashing 원리 정리

- **할 일**: `day55/` 안에 `consistent_hashing_notes.md`를 만든다. **해시 링** 위에 키와 노드를 배치하고, **시계 방향 다음 노드**가 그 키를 담당. **가상 노드**로 분포 균등화. 2~3문장으로 적는다.
- **완료 조건**: "키/노드를 링에 매핑, 담당 노드 = 시계 방향 다음"이 설명되어 있다.

### 2. Consistent Hashing 구현 + 실험

- **할 일**: `day55/` 안에 `consistent_hashing.py`를 만든다. **해시 함수**(예: hash % 360 또는 해시를 0~1 범위로), **노드 추가**(링에 노드 추가), **노드 제거**, **get_node(key)**(키가 어느 노드에 할당되는지 반환). **가상 노드**는 노드당 여러 점을 링에 두면 됨. `if __name__ == "__main__"`: 노드 3개일 때 키 10개 할당 → 노드 1개 제거 → 같은 키 10개가 어떻게 재할당되는지 출력. "이동하는 키 비율"을 주석으로 적어 둔다.
- **완료 조건**: 노드 추가/제거 시 get_node 결과가 바뀌고, 재할당되는 키만 이동함을 확인했다.
