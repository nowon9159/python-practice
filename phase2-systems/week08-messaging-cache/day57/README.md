# Day 57: 설계 실습 — URL 단축기 설계

## 🎯 오늘의 목표

- **URL 단축기**를 **요구사항 → 다이어그램 → API → 스키마** 순으로 설계하고, (선택) 단축 URL 생성 로직을 코드로 구현한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 요구사항·High-level 다이어그램

- **할 일**: `day57/` 안에 `url_shortener_design.md`를 만든다. **요구사항**: 짧은 URL 생성, 리다이렉트, (선택) TTL. **High-level**: 클라이언트 → API 서버 → DB, (선택) 캐시. 다이어그램은 ASCII나 Mermaid로 컴포넌트 3~4개.
- **완료 조건**: 요구사항 3개 이상, 다이어그램 1개가 문서에 있다.

### 2. API·DB 스키마

- **할 일**: 같은 문서에 **API** 2개 이상 (예: POST /shorten, GET /:id 리다이렉트). **DB 스키마**: short_url(또는 id), long_url, created_at 등 컬럼 3개 이상.
- **완료 조건**: API와 스키마가 문서에 명시되어 있다.

### 3. (선택) 단축 URL 생성 코드

- **할 일**: `day57/shorten.py`에 **long_url → short code** 생성 로직을 구현한다 (해시 후 base62 인코딩, 또는 UUID 일부). 충돌 시 재시도 한 줄이라도 적어 둔다.
- **완료 조건**: 선택. 구현했다면 long_url 입력에 대해 short code가 나온다.

### 4. Anki 카드 추가

- **할 일**: "URL 단축기 설계: 단축 코드 생성, 리다이렉트, 스키마" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
