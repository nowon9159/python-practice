# Day 47: Write-Ahead Log (WAL) 구현

## 🎯 오늘의 목표

- **WAL** 개념을 정리하고, **put 시 로그에 먼저 append 후 메모리 반영**, **재시작 시 로그 재생**으로 복구하는 코드를 작성한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. WAL 쓰기/재생 흐름 정리

- **할 일**: `day47/` 안에 `wal_notes.md`를 만든다. **WAL**: 변경을 **로그 파일에 먼저 append**한 뒤 실제 데이터 구조를 갱신. 장애 시 **로그 재생(redo)**으로 복구. 2~3문장으로 적는다.
- **완료 조건**: "먼저 로그, 그 다음 데이터 / 재시작 시 재생"이 설명되어 있다.

### 2. WAL 구현

- **할 일**: `day47/` 안에 `wal_kv.py`를 만든다. **put(key, value)**: (1) 로그 파일에 "put,key,value" 형태로 append. (2) 메모리 dict에 key→value 반영. **get(key)**: 메모리에서 조회. **재시작 시**: 로그 파일을 처음부터 읽어 put 연산만 재적용해 메모리 dict를 복구. `if __name__ == "__main__"`에서 put 여러 번 → (프로세스 재시작 시뮬레이션: 새 인스턴스에서 로그만 읽고 복구) → get으로 값이 복구되는지 확인.
- **완료 조건**: put 후 로그에 기록되고, "재시작" 후 로그만으로 get이 복구된 값을 반환한다.

### 3. Anki 카드 추가

- **할 일**: "WAL: 먼저 로그 기록, 재시작 시 redo" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
