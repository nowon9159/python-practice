# Day 49: 쿼리 최적화 — EXPLAIN 분석 + 인덱스 설계

## 🎯 오늘의 목표

- **EXPLAIN**(또는 EXPLAIN ANALYZE)으로 실행 계획을 읽고, **인덱스 설계**를 한 번 경험한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. EXPLAIN 실행·해석

- **할 일**: 실제 DB(PostgreSQL/MySQL/SQLite 등)에서 **EXPLAIN** 또는 **EXPLAIN ANALYZE**를 실행한다. `day49/` 안에 `explain_notes.md`를 만들고, **Seq Scan vs Index Scan**, **비용 추정** 등 출력 항목 중 2~3개를 해석해 적는다. (DB 없으면 공식 문서로 EXPLAIN 출력 예시를 보고 해석만 해도 됨.)
- **완료 조건**: EXPLAIN 출력의 의미를 2문장 이상 설명할 수 있다.

### 2. 인덱스 설계 연습

- **할 일**: "느린 쿼리" 시나리오 하나를 가정한다(예: WHERE name = ? 자주 호출). **어떤 컬럼에 인덱스를 걸면 좋을지** 제안하고, (가능하면) 인덱스 추가 전·후 EXPLAIN을 비교해 **비용 또는 스캔 방식 변화**를 메모한다. `day49/` 에 `index_design.md`로 정리.
- **완료 조건**: 인덱스 제안 1개와 (가능 시) 전/후 비교가 문서에 있다.

### 3. Anki 카드 추가

- **할 일**: "EXPLAIN: Seq Scan vs Index Scan / 인덱스 설계 포인트" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
