## ✅ 오늘 할 일 (순서대로)

### 1. 격리 수준·MVCC 정리

- **할 일**: `day48/` 안에 `mvcc_notes.md`를 만든다. **Read Uncommitted / Read Committed / Repeatable Read / Serializable** 차이를 표 또는 2~3문장으로 적는다. **MVCC**: 여러 버전을 유지해 읽기는 스냅샷, 쓰기 충돌은 별도 처리. Dirty read, Phantom read 예시 한 줄씩.
- **완료 조건**: 4가지 격리 수준과 MVCC를 말로 구분할 수 있다.

### 2. (선택) MVCC 시뮬레이터

- **할 일**: `day48/` 안에 `mvcc_sim.py`를 만든다. 트랜잭션별 "보이는" **버전**(스냅샷)을 관리하고, **commit** 시 반영, **abort** 시 버리기. 단순화: 키 하나, 버전 리스트만 있어도 됨.
- **완료 조건**: 선택. 구현했다면 트랜잭션 2개가 서로 다른 버전을 보는 상황을 한 번 확인한다.
