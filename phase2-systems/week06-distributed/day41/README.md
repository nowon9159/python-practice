# Day 41: 분산 KV 스토어 Day1 — 단일 노드 → 복제 구조

## 🎯 오늘의 목표

- **단일 노드 KV** 스토어(get/set/delete)를 구현하고, **복제 구조**(2~3 노드, 쓰기 라우팅)를 설계·최소 구현한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 단일 노드 KV 구현

- **할 일**: `day41/` 안에 `kv_store.py`를 만든다. **get(key)**, **set(key, value)**, **delete(key)** 를 지원하는 클래스. 저장은 **in-memory dict** 또는 (선택) 파일 한 개로 persist. `if __name__ == "__main__"`에서 set → get → delete → get으로 동작 확인.
- **완료 조건**: 단일 노드에서 get/set/delete가 기대대로 동작한다.

### 2. 복제 구조 설계 및 최소 구현

- **할 일**: 같은 파일 또는 `kv_replicated.py`에 **노드 2~3개**를 만든다. 각 노드는 위 KV 스토어를 가진다. **쓰기**는 (규칙을 정해) **리더 노드**에만 보내고, (선택) 리더가 팔로워에 복제하는 로직을 stub으로라도 둔다. **읽기**는 어떤 노드에서든 허용하거나, 리더만 허용하도록 규칙을 정한다. "쓰기 → 리더, 읽기 → 아무 노드 또는 리더" 흐름이 코드에 드러나면 됨.
- **완료 조건**: 복제 구조에서 쓰기 요청이 리더로 가고, 읽기가 한 노드라도 동작한다. (Day 42에서 Raft 연동 예정)

### 3. (선택) Anki

- **할 일**: "분산 KV: 단일 노드 → 리더에 쓰기, 복제" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
