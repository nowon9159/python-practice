# Day 36: 프로파일링 — cProfile, memory_profiler, py-spy

## 🎯 오늘의 목표

- **cProfile**으로 스크립트의 CPU 병목을 찾고, **한 가지 이상 최적화**를 적용한 뒤 결과를 비교한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. cProfile으로 병목 찾기

- **할 일**: **대상 스크립트**를 하나 정한다 (Day 2 Fibonacci, Day 4 해시 문제, 또는 반복 루프가 많은 임의 스크립트). `day36/` 안에 `profile_target.py`를 두거나 기존 코드를 복사해 둔다. 터미널에서 **`uv run python -m cProfile -s cumulative profile_target.py`** (또는 `-o profile.out` 후 `pstats`로 확인)를 실행한다. **출력에서 누적 시간이 큰 함수 1~3개**를 골라 `day36/profile_result.md`에 "병목 후보: 함수명, 누적 시간" 형태로 적는다.
- **완료 조건**: cProfile 출력을 한 번 확인했고, 병목 후보 1개 이상을 문서에 적었다.

### 2. 한 가지 이상 최적화 적용

- **할 일**: 위에서 고른 **병목 후보 1개**에 대해, **가설**을 세우고 수정한다 (예: 불필요한 반복 제거, 캐시 도입, 알고리즘 변경). 수정 후 **같은 입력으로 cProfile을 다시 실행**해 누적 시간이 줄었는지 확인한다. 결과를 `profile_result.md`에 "최적화 내용, 전/후 시간(또는 호출 횟수)"로 추가한다.
- **완료 조건**: 최적화 전·후 비교가 문서에 남아 있다.

### 3. (선택) memory_profiler 또는 py-spy

- **할 일**: **memory_profiler**: `@profile` 데코레이터나 `python -m memory_profiler script.py`로 메모리 사용량을 확인해 보기. 또는 **py-spy**: 실행 중인 프로세스에 샘플링으로 attach해 보기. 도구 설치·실행 방법을 1~2줄 메모해 둔다.
- **완료 조건**: 선택. 사용해 봤으면 "용도 한 줄"을 메모했다.

### 4. Anki 카드 추가

- **할 일**: "cProfile: CPU 병목 찾기, cumulative 시간으로 호출 트리 분석" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.
