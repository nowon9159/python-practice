# Day 33: Socket 프로그래밍 — TCP 서버·클라이언트

## 🎯 오늘의 목표

- **TCP** 소켓으로 **에코 서버**와 **에코 클라이언트**를 구현하고, 서로 통신이 되는지 확인한다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. TCP 에코 서버 구현

- **할 일**: `day33/` 안에 `tcp_echo_server.py`를 만든다.
  - **동작**: 한 포트(예: 9999)에 **bind** → **listen** → **accept** 루프. accept로 받은 클라이언트 소켓에서 **recv**한 데이터를 그대로 **send**로 돌려준다(에코). 한 클라이언트 처리 후 다음 accept (순차). (선택) `threading`으로 클라이언트마다 스레드 생성해 동시 처리.
  - **종료**: Ctrl+C 또는 고정 횟수 accept 후 종료.
- **완료 조건**: 서버를 실행한 뒤 클라이언트로 문자열을 보내면 같은 문자열이 돌아온다.

### 2. TCP 에코 클라이언트 구현

- **할 일**: 같은 폴더에 `tcp_echo_client.py`를 만든다.
  - **동작**: **connect**(서버 주소, 포트) → **send**(바이트) → **recv** → 수신한 데이터 출력. 인자 또는 하드코딩으로 서버 주소·포트 지정.
  - **실행 예**: `uv run python tcp_echo_client.py` (서버가 먼저 떠 있는 상태).
- **완료 조건**: 서버 실행 후 클라이언트를 실행했을 때 보낸 메시지가 그대로 출력된다.

### 3. (선택) 여러 클라이언트 연결 테스트

- **할 일**: 클라이언트를 2개 이상 띄우거나, 순차적으로 여러 번 연결해 보기. 서버가 순차 처리면 한 번에 한 연결만, threading이면 동시 연결 가능.
- **완료 조건**: 동작 방식을 주석으로 남겼다.

### 4. Anki 카드 추가

- **할 일**: "TCP 서버 흐름: socket → bind → listen → accept(루프) → recv/send" **카드 1장** 추가.
- **완료 조건**: Phase 2 덱에 반영했다.

## 📌 참고 (복습할 개념)

- 서버: socket → bind → listen → accept. 클라이언트: socket → connect → send/recv.
- 바이트 스트림, 버퍼 크기, 에러 처리.
