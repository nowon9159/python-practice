# Day 35: 메모리 관리 — 참조 카운팅, gc, __slots__

## 🎯 오늘의 목표

- 파이썬 **참조 카운팅**과 **순환 참조·gc**를 정리하고, **gc 모듈** 사용법과 **__slots__** 예제를 코드로 남긴다.

---

## ✅ 오늘 할 일 (순서대로)

### 1. 참조 카운팅·순환 참조 정리

- **할 일**: `day35/` 안에 `memory_notes.md`를 만든다. **참조 카운팅**이 무엇인지(객체마다 refcount, 0이 되면 회수), **순환 참조**가 무엇인지(A→B→A 등으로 refcount가 0이 안 됨), **gc**가 순환 참조를 어떻게 처리하는지 **2~3문장**으로 적는다.
- **완료 조건**: "refcount 0 → 회수", "순환 참조는 gc가 찾아서 회수"가 설명되어 있다.

### 2. gc 모듈 실험

- **할 일**: `day35/` 안에 `gc_experiment.py`를 만든다. **gc.get_objects()** 로 현재 추적 중인 객체 개수 출력, **gc.collect()** 호출 전후 비교 (선택). **sys.getrefcount(obj)** 로 특정 객체의 참조 횟수 확인 (주의: getrefcount에 넘기는 순간 +1). 순환 참조 예제(두 클래스가 서로를 참조)를 만들고, gc 수집 전후 객체 수를 출력해 보기.
- **완료 조건**: 스크립트 실행 시 gc 관련 출력이 나오고, refcount 또는 collect 결과를 주석으로 해석해 뒀다.

### 3. __slots__ 예제 (선택)

- **할 일**: 같은 폴더에 `slots_example.py`를 만든다. **일반 클래스**(인스턴스 __dict__ 있음)와 **__slots__ = ("a", "b")** 로 같은 필드만 가진 클래스를 정의하고, (선택) **메모리 사용량**을 비슷한 개수 인스턴스로 비교해 보기 (예: `sys.getsizeof` 또는 memory_profiler).
- **완료 조건**: __slots__를 쓴 클래스가 __dict__가 없음을 확인했고, (선택) 메모리 차이를 한 줄 메모했다.
